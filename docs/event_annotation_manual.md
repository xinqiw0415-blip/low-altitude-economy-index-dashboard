# 低空经济事件标注手册 v1.0

## 标注单位

一个事件必须对应明确的行为主体、发布日期和可引用证据。政策全文原则上先标注“发布事件”，若同一文件包含独立且可执行的补贴、空域改革或基础设施任务，可拆分为子事件。

## 核心字段

- `event_type`：以事件主要作用机制分类，不按标题机械分类。
- `direction`：对低空经济产业预期的方向，不等同于文本语气。
- `intensity`：1为倡议性表述，3为有明确任务和期限，5为法规或包含重大资金、强制制度安排。
- `uncertainty`：1为责任主体、期限、资金均明确；5为原则性愿景且实施条件高度不确定。
- `novelty`：相对既有国家和本地区政策的新信息程度。
- `evidence_span`：必须复制能够支持分类的最短原文，不允许使用模型补充知识。

## 事件类型

| 类型 | 定义 |
|---|---|
| `strategic_plan` | 中长期规划、行动方案、发展目标 |
| `support_measure` | 综合性扶持措施 |
| `regulation` | 法规、飞行管理、准入与监管制度 |
| `airspace_reform` | 空域划设、审批流程和飞行服务改革 |
| `infrastructure` | 起降点、机场、通信导航监视和数字底座建设 |
| `fiscal_subsidy` | 补贴、奖励、贴息和税费支持 |
| `industry_standard` | 统计分类、技术标准和行业规范 |
| `insurance_finance` | 保险、基金、融资与金融支持 |

## 双人标注流程

1. 两名标注员独立标注同一批文档，不查看对方结果。
2. 类别变量计算 Cohen's Kappa，强度变量计算加权Kappa或ICC。
3. Kappa低于0.75的字段重新讨论定义并返标。
4. 分歧由第三人或项目负责人裁决，保留原始标注和裁决记录。
5. 训练、验证、测试按文档划分，禁止同一政策的正文与解读跨集合泄漏。

## 时间对齐

网页只有日期而无准确时刻时，一律将事件信息集对齐到“发布日期后的第一个交易日”。这是保守规则，可防止将盘后信息错误用于当天收益预测。
